Layerfig accepts a few configuration file formats.


## JSON

To overcome JSON's limitations (e.g., lack of comments), the library supports `.json`, `.jsonc`, and `.json5` formats interchangably:


<Tabs>
    <TabItem label="config.ts">

```ts
export const config = new ConfigBuilder({
  validate: (finalConfig) => schema.parse(finalConfig)
})
    .addSource("base.json")
    .addSource("local.jsonc")
    .addSource("prod.json5")
    .build();
```

    </TabItem>

    <TabItem label="base.json">

```json
{
  "appURL": "http://localhost:3000"
}
```

    </TabItem>

    <TabItem label="local.jsonc">

```jsonc
{
  // Comment
  "appURL": "http://localhost:4321"
}
```

    </TabItem>

    <TabItem label="prod.json5">

```jsonc
{
  // Comment
  appURL: "http://my-app.com"
}
```

    </TabItem>
</Tabs>



## YAML

Both `.yaml` and `.yml` file extensions are supported.

<Tabs>
    <TabItem label="config.ts">

```ts
export const config = new ConfigBuilder({
  validate: (finalConfig) => schema.parse(finalConfig)
})
  .addSource("base.yml")
  .addSource("prod.yaml")
  .build();
```

    </TabItem>

    <TabItem label="base.yaml">

```yml
appURL: http://localhost:3000
```

    </TabItem>

    <TabItem label="prod.yaml">

```yaml
appURL: http://my-app.com
```

    </TabItem>
</Tabs>
